# GitHub Secrets Required for CI/CD Pipeline
# Go to: GitHub Repo -> Settings -> Secrets and variables -> Actions -> New repository secret
#
# ONLY 2 SECRETS NEEDED — everything else is auto-created by CloudFormation!

AWS_ACCESS_KEY_ID=<your-aws-access-key>
AWS_SECRET_ACCESS_KEY=<your-aws-secret-key>

# Optional (for SonarCloud integration):
# SONAR_TOKEN=<from sonarcloud.io>
# SONAR_ORG=<your-sonarcloud-org>

# All other resources (EC2, RDS, S3, CloudFront, IAM, SSH Keys) are
# automatically provisioned by the CloudFormation template in
# cloudformation/infrastructure.yml
#
# HOW IT WORKS:
# 1. Add the 2 AWS secrets above
# 2. Go to Actions tab -> "Setup AWS Infrastructure" -> Run workflow
# 3. Wait ~15 min for all resources to be created
# 4. Push code to main -> CI/CD pipeline auto-deploys
#
# NEVER commit real values to the repository!
